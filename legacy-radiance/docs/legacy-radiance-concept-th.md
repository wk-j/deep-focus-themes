# Legacy Radiance — แนวคิดชุดสี

## ที่มา

ภาพนี้คือภาพ 3D ที่สร้างขึ้นสำหรับคลังภาพ แสดงซอร์สโค้ด COBOL ลอยอยู่บนระนาบที่ถอยลึกเข้าไปในมุมมองทับพื้นหลังสีดำสนิท — ลักษณะแบบที่ใช้ประกอบบทความเรื่อง "ระบบเก่า" ในนิตยสารธุรกิจ แต่ถ้ามองผ่านลายน้ำออกไป จะพบสีแสงสามความถี่ที่ปรากฏพร้อมกัน ได้แก่ สีเขียวฟอสฟอร์บนแผงด้านซ้ายใกล้ (`#00C840`) ซึ่งเป็นสีเขียวเฉพาะของสารเคลือบฟอสฟอร์ P1 บนเทอร์มินัลยุค 1970s อย่าง Hazeltine หรือ ADM-3A; สีน้ำเงินโคบอลต์เข้มบนระนาบกลาง (`#2A7EE8`) สีของแสงพื้นหลังแคโทดเย็น; และสีฟ้าไซยานไอออนบนด้านขวาและตำแหน่งเคอร์เซอร์ (`#00D4C8`) ซึ่งเป็นการแผ่รังสีที่มีพลังงานสูงสุดในช่วงแสงที่ตามองเห็น ก่อนถึงสีน้ำเงิน-ม่วง สีของเทอร์มินัลที่กำลังปรับระดับคอนทราสต์จนเกือบล้างภาพ พื้นหลังไม่ใช่แค่มืด — มันคือการไม่มีแสงสะท้อนใด ๆ ในช่องว่างระหว่างระนาบที่เปล่งแสงในตัวเอง

---

## วิสัยทัศน์

Legacy Radiance เป็นชุดสีสำหรับเขียนโค้ดที่สร้างขึ้นจากฟิสิกส์จริงของการเปล่งแสงฟอสฟอร์ใน CRT (จอภาพรังสีแคโทด) ความตึงเครียดหลักคือระหว่างสิ่งเก่าแก่และสิ่งที่ยังมีพลังงานไฟฟ้า: โค้ดที่เขียนก่อนนักพัฒนาส่วนใหญ่จะเกิด ยังคงแผ่แสงเย็นเข้าไปในห้องที่ไม่มีแสงใด ๆ

แก้ไขความตึงเครียดนี้ผ่านการตัดสินใจออกแบบสามประการ:

- **พื้นดำสนิท** — พื้นหลัง (`#080808`) ไม่ใช่สีเทาเข้มหรือสีดำเกือบสนิทที่มีโทนสี มันคือสีของช่องว่างระหว่างระนาบในภาพ — ไม่ใช่ความว่างเปล่า แต่เป็นความไม่มีแสง แถบด้านข้างลึกลงไปเป็น `#040404` ซึ่งเป็นความแตกต่างที่ให้ความรู้สึกเป็นความลึกมากกว่าการเปลี่ยนสี
- **การเปล่งแสงสามความถี่** — ชุดสีเน้นสร้างขึ้นบนความถี่ฟอสฟอร์สามแบบพอดี: สีเขียว (`#00C840`) สำหรับฟังก์ชัน สิ่งที่ทำงาน; โคบอลต์ (`#2A7EE8`) สำหรับคีย์เวิร์ด คำสั่งโครงสร้างที่ไม่เปลี่ยนแปลงมาสี่สิบปี; และไซยานไอออน (`#00D4C8`) สำหรับชนิดข้อมูล ชั้นขอบเขตที่นิยามข้อมูล แต่ละความถี่มีสารเคมีที่ต่างกัน — พวกมันไม่ผสมกันที่ระดับความอิ่มตัวกลาง
- **ลำดับชั้นการสลาย** — ข้อความหน้าจอ (`#C8E0D0`) คือสีของแสงเรืองจากจอฟอสฟอร์ที่มองผ่านกระจก CRT ที่เป็นฝุ่นเล็กน้อย — ไม่ใช่สีขาว ไม่ใช่สีเขียว แต่เป็นสีขาวที่เอียงไปทางเขียวของจอที่เปิดทิ้งไว้หลายชั่วโมง คอมเมนต์ (`#304838`) คือฟอสฟอร์ตาย: สีของพื้นที่หน้าจอที่ถูกเขียนทับซ้ำจนสารเคลือบเริ่มสึกหรอ

ผลลัพธ์คือพื้นที่ทำงานที่ให้ความรู้สึกเหมือนนั่งอยู่หน้าเทอร์มินัลในศูนย์ข้อมูลตอนตีสาม — แหล่งกำเนิดแสงเดียวในห้องคือตัวโค้ดเอง

## หลักการออกแบบ

### 1. ความถี่ฟอสฟอร์ในฐานะบทบาทไวยากรณ์

จอ CRT ใช้สารฟอสฟอร์ต่างชนิดเพื่อสร้างสีต่างกัน — P1 สำหรับสีเขียว, P4 สำหรับสีขาว, P31 สำหรับสีเขียวที่เอียงไปทางน้ำเงิน สารแต่ละชนิดเปล่งแสงที่ความยาวคลื่นจำเพาะเมื่อถูกยิงด้วยอิเล็กตรอน Legacy Radiance จัดบทบาทไวยากรณ์ตามความถี่: ฟอสฟอร์พลังงานต่ำสุดที่ตามองเห็น (สีเขียว, `#00C840`) สำหรับประเภทโทเค็นที่พบบ่อยที่สุด (ฟังก์ชัน), ความถี่กลาง (น้ำเงินโคบอลต์, `#2A7EE8`) สำหรับจุดยึดโครงสร้าง (คีย์เวิร์ด), และการเปล่งแสงต่อเนื่องที่มีพลังงานสูงสุด (ไซยาน, `#00D4C8`) สำหรับข้อมูลชนิด นี่ไม่ใช่ความบังเอิญ — ฟังก์ชันสีเขียวถูกเรียกมากที่สุด คีย์เวิร์ดสีน้ำเงินถูกอ่านมากที่สุด ชนิดข้อมูลสีไซยานถูกตรวจสอบเมื่อมีสิ่งผิดปกติ ความถี่สอดคล้องกับรูปแบบการใช้งาน

### 2. ระนาบคำสั่ง กับ ระนาบข้อมูล

ภาพแสดงสองชั้นทางสายตาที่ต่างกัน: ระนาบสีเขียวใกล้ (เชิงปฏิบัติการ, เชิงสั่งการ — สิ่งที่โปรแกรมทำ) และระนาบสีน้ำเงิน/ไซยานไกล (เชิงโครงสร้าง, เชิงประกาศ — สิ่งที่โปรแกรมเป็น) สิ่งนี้แมปตรงกับการแบ่งคีย์เวิร์ด/ฟังก์ชัน คีย์เวิร์ดโคบอลต์ (`#2A7EE8`) คือระนาบคำสั่ง — `IF`, `PERFORM`, `EXEC`, `END-IF` ในภาพ กริยาที่ไม่เปลี่ยนแปลงนับตั้งแต่มีการกำหนดมาตรฐาน COBOL ฟังก์ชัน Phosphor Green (`#00C840`) คือระนาบข้อมูล — โพรซีเดอร์ที่มีชื่อ สิ่งที่ถูกเรียก สิ่งเหล่านี้สร้างลำดับชั้นทางสายตาที่ชัดเจน:

| บทบาท | สี | Hex | เหตุผล |
|---|---|---|---|
| คีย์เวิร์ด | Cobalt | `#2A7EE8` | ระนาบสีน้ำเงินกลางในภาพ — โครงสร้าง หนักแน่น มีอำนาจ |
| ฟังก์ชัน | Phosphor Green | `#00C840` | การเปล่งแสงฟอสฟอร์ P1 — สีเขียวเทอร์มินัลที่พบบ่อยที่สุด สีแห่งการทำงาน |
| ชนิดข้อมูล | Ionized Cyan | `#00D4C8` | การเปล่งแสงที่จุดประจุสูงสุด — สีต่อเนื่องสว่างที่สุด ทำเครื่องหมายขอบเขตชนิดข้อมูล |
| สตริง | Plasma | `#00F0A0` | สีเขียวที่เอียงไปทางไซยาน — ข้อมูลระหว่างเดินทาง อยู่ระหว่างสองระนาบ |
| ค่าคงที่ | Phosphor White | `#D8F0E0` | แสงหน้าจอที่ความสว่างสูงสุด — สิ่งที่ไม่เปลี่ยนแปลง |
| ตัวเลข | Pale Green | `#80C898` | ฟอสฟอร์ประจุครึ่ง — เชิงตัวเลข แม่นยำ เงียบกว่าสีเขียวเต็มที่ |
| ตัวแปร | Screen Glow | `#C8E0D0` | แสงสะท้อนของหน้าจอที่มองผ่านกระจกเป็นฝุ่น |
| คอมเมนต์ | Dead Phosphor | `#304838` | สารเคลือบสึกหรอ — ยังอยู่แต่ไม่เปล่งแสงที่ความเข้มเต็มอีกต่อไป |

### 3. พื้นหลังในฐานะเวที ไม่ใช่ความว่างเปล่า

ความมืดในธีมนี้ไม่ใช่เพื่อตกแต่ง มันคือความมืดเฉพาะของห้องที่มีแสงเดียวคือโค้ด

- **พื้นผิวตัวแก้ไข** (`#080808`) — สูงกว่าสีดำสนิทแปดระดับ ขั้นต่ำที่จำเป็นเพื่อหลีกเลี่ยงการแสดงผลผิดพลาดบนบางหน้าจอ นี่คือช่องว่างระหว่างระนาบโค้ดลอยในภาพ
- **แผง/แถบด้านข้าง** (`#040404`) — มืดกว่าตัวแก้ไขสองระดับ ความแตกต่างไม่ปรากฏเป็นสี แต่รับรู้ได้เป็นความลึก แผงรู้สึกอยู่ไกลกว่าเพราะมันอยู่ไกลกว่าจริง ๆ
- **การไฮไลต์** (`#0E1A10`) — สีเขียวเข้มอ่อน ๆ ราวกับว่าพื้นที่ที่เลือกได้รับแสงจากการเปล่งแสงฟอสฟอร์ที่อยู่ใกล้ ๆ แทนที่จะเป็นสีโอเวอร์เลย์แบบแบน
- **ข้อผิดพลาด** ใช้ Overload Red (`#E83840`) — ไม่ใช่สีแดงเตือนภัยอิ่มตัวสูงของ UI สมัยใหม่ แต่เป็นสีแดง-ส้มเฉพาะของ CRT ที่ถูกขับเกินแรงดันที่ระบุ: ยังคงเป็นสีแดง แต่มีความอุ่นของสิ่งที่กำลังจะพัง ไม่ใช่สิ่งที่พังไปแล้ว

### 4. เคอร์เซอร์ในฐานะจุดยึด

เคเรตใช้ Ionized Cyan (`#00D4C8`) ในภาพ บล็อกเคอร์เซอร์เรืองแสงสีไซยาน — สีเขียว-น้ำเงินเฉพาะของเคอร์เซอร์เทอร์มินัลที่ความเข้มสูงสุด บล็อกที่บอกคุณว่าอักขระถัดไปจะอยู่ที่ไหน บนเทอร์มินัลจริง เคอร์เซอร์คือจุดสว่างที่สุดบนหน้าจอเสมอ ใน Legacy Radiance มันทำหน้าที่เดียวกัน: `#00D4C8` เป็นสีที่มีพลังงานสูงที่สุดในชุดสีที่ไม่ได้ใช้สำหรับบทบาทไวยากรณ์หลัก ซึ่งหมายความว่าเคอร์เซอร์ไม่มีวันถูกสับสนกับโค้ดและหาพบได้เสมอในทุกบริบท

## การจับคู่แบบอักษร

ชุดสี "Legacy Radiance" จับคู่กับ:

- **หลัก:** JetBrains Mono — รูปแบบตัวอักษรที่ยืดเล็กน้อยในขนาดเล็กนึกถึงตัวอักษรบิตแมปอ้วนของเทอร์มินัล 80 คอลัมน์ สีเขียวฟอสฟอร์อ่านง่ายที่สุดผ่านช่องเปิดที่กว้างขวาง
- **ทางเลือก:** Fira Code — รูปลิเกเจอร์สำหรับ `=>` และ `->` สะท้อนถึงวิธีที่เส้นทางตัวแปร COBOL เขียนด้วยขีดกลาง เป็นเสียงสะท้อนทางสายตาจากวัสดุต้นทางโดยไม่ต้องเป็นตัวอักษร

ลิเกเจอร์ในธีมนี้อ่านเป็นทางลัดที่แกะสลักลงในสิ่งที่ไม่ได้ออกแบบมาสำหรับทางลัด — พวกมันรู้สึกเหมือนการเพิ่มประสิทธิภาพที่โปรแกรมเมอร์ทำหลังจากหลายปีบนระบบเดียวกัน ขัดเกลาจนเรียบ

## ความรู้สึกในทางปฏิบัติ

จินตนาการว่าคุณกำลังเปิดไฟล์ SQL migration ตอนตีสองในศูนย์ข้อมูลที่ดับไฟฟลูออเรสเซนต์ด้านบนทั้งหมด พื้นหลังคือความว่างเปล่า — ไม่ใช่ความมืดที่สะดวกสบายของ dark mode สมัยใหม่ แต่คือความมืดของห้องที่จอแสดงผลเป็นแหล่งโฟตอนเดียวในระยะหกเมตรรอบข้าง คีย์เวิร์ดสีน้ำเงินโคบอลต์ (`#2A7EE8`) ปรากฏก่อน หนักและเป็นโครงสร้าง: `SELECT`, `FROM`, `WHERE` — คำสั่งเดียวกับที่พิมพ์เข้าเทอร์มินัลมาตั้งแต่ก่อนคุณเกิด แสดงผลด้วยสีน้ำเงินเย็นเดียวกับที่ระนาบกลางลอยอยู่ ฟังก์ชันผุดขึ้นมาด้วยสีเขียวฟอสฟอร์ (`#00C840`) การเปล่งแสงฟอสฟอร์ P1 ที่โปรแกรมเมอร์ที่เคยใช้ VT100 รู้จักโดยไม่ต้องตั้งชื่อ — มันคือสีเขียว สีเขียวดั้งเดิม สีเขียวที่หมายความว่าเครื่องกำลังทำงาน การระบุชนิดข้อมูลเรืองแสงด้วยไซยานไอออน (`#00D4C8`) ความถี่สูงกว่าสีเขียวของฟังก์ชัน ทำเครื่องหมายขอบเขตของสิ่งที่ข้อมูลได้รับอนุญาตให้เป็น สตริงรวมตัวเป็นสีเขียวพลาสมา (`#00F0A0`) เอียงไปทางไซยานเล็กน้อย ข้อมูลระหว่างเดินทางระหว่างระนาบปฏิบัติการและโครงสร้าง ตัวเลขอ่านด้วยสีเขียวอ่อน (`#80C898`) ประจุครึ่ง แม่นยำ ตัวแปรคือแสงหน้าจอ (`#C8E0D0`) — สีขาวเกือบขาวของหน้าจอที่เปิดทิ้งไว้สี่ชั่วโมง ไม่ใช่ขาวสะอาด แต่เป็นขาวที่ถูกย้อมโดยทุกสิ่งรอบข้าง คอมเมนต์คือฟอสฟอร์ตาย (`#304838`) ยังอยู่ ยังอ่านได้ถ้าเงยดู แต่ไม่เปล่งแสงในระดับของโค้ดที่ยังทำงาน เคอร์เซอร์ของคุณลุกโชนด้วยสีไซยานไอออน คุณรู้ว่าคุณอยู่ที่ไหนเสมอ เพราะมันคือจุดสว่างที่สุดในห้อง
